## 三、用户界面功能

### 3.1 终端用户操作界面

#### 3.1.1 流程选择

- 选择新流程或恢复中断流程  
- 选择实验流程文件

![流程选择](media/image3.png)

#### 3.1.2 测试前准备

- 系统自动提醒用户进行必要的准备工作（水箱是否有水，废液桶是否已满，排毒桶是否已满），有异常时弹窗交互

#### 3.1.3 样本区域扫描

- 扫描样本区二维码  
- 自动统计玻片样本数量  
- 扫描头安装在机械臂上，机械臂移动到相应位置扫码

![样本扫描界面](media/image4.png)

#### 3.1.4 试剂区域扫描

- 试剂共 5 列试剂架，每列 8 个瓶子（共 40 个试剂位，编号 R1~R40）  
- 瓶子上有条码，试剂架对应位置也有条码，识别试剂放置位置  
- 试剂区有独立的固定扫码头（左侧），试剂架插入时自动扫描  
- 每个试剂架有传感器识别是哪一列的架子  
- 当检测到样本所需试剂缺失时，系统自动提示

![试剂扫描界面](media/image6.png)  
![试剂缺失提示](media/image7.png)

#### 3.1.5 运行界面

- 实时显示流程操作进度、运行步骤、运行状态  
- 显示玻片号码、液位信息等关键数据  
- 用户可执行：
  - 暂停 / 继续  
  - 终止运行  
  - 中途添加样本（选择流程名称，免疫组化需添加一抗代码）  
- 系统智能处理：
  - 试剂不足时提醒添加  
  - 故障时可从大步骤重新开始  
  - 根据流程自动添加一抗、二抗及其他通用试剂

![运行状态页面](media/image8.png)

- **试剂管理**  
  - 支持每种试剂的用量、剩余量记录，设置最低报警功能  
  - 双套二抗试剂上载，机器运行期间可随时更换或上载新试剂而无需停机，实现随到随检

### 5.3 通道移位配置

- 用于配置各个加样孔位的坐标位置，包含：
  - 33 个试剂位（R1~R33，实际共 40 位，此处为示意）  
  - 洗站 2 个（内壁和外壁）  
  - 16 个样品位  
- 试剂区共 5 列，每列 8 个，配置 X/Y 轴时，定位每列第一个（R1/R12/R23）和最后一个（R11/R22/R33），其余坐标按线性插值计算  
  - 例如：R2 = R1 + 2×(R11 - R1)/10  
- 每个孔位的 Z 轴配置信息：
  - **Z-Travel**：机械移动安全高度  
  - **Z-Start**：探液高度（慢速移动准备探液）  
  - **Z-End**：针尖最大下降深度  
  - **Z-Dispens**：吸液/排液高度  
  - **液体类型**：孔位所关联的液体类型

### 5.4 液体类型配置

Liquid Class 是一组预设或自定义的移液参数集合，针对某类液体的物理特性（粘度、表面张力、挥发性等）优化移液动作。

![液体类型配置示意](media/image15.png)

| **吸液参数**                  | **作用**                                                         |
| ----------------------------- | ---------------------------------------------------------------- |
| 探液下降速度                  | 控制吸头探测液面时的下降速度，影响液位检测精度                   |
| 液面跟随深度                  | 吸头在探测到液面后继续浸入的深度                                 |
| 吸液速度                      | 柱塞吸取液体的速度                                               |
| 吸液前延时                    | 吸头到位后、开始吸液前的等待时间，使系统稳定                     |
| 吸液后延时                    | 吸液完成后、提针前的停留时间                                     |
| 回撤速度                      | 吸液后吸头从液体中向上移动的速度                                 |
| System Trailing Air Gap       | 在系统液与样品之间保留的空气段，隔离样品与泵/阀                  |
| Leading Air Gap               | 吸液前先吸入的空气段，位于系统空气与样品之间                     |
| Trailing Air Gap              | 吸液后在样品后方吸入的空气段，包裹样品液柱                       |
| Excess Volume                 | 额外多吸一定体积，确保最后一次分液体积准确                       |
| Conditioning Volume           | 调节量                                                           |

| **加液参数**                              | **作用**                                                                 |
| ----------------------------------------- | ------------------------------------------------------------------------ |
| 加液速度（Dispense Speed）                | 控制柱塞推出液体的速度                                                   |
| 断液速度（Breakoff Speed）                | 排液结束时急减速或反向微动，干净切断液流，防止挂滴或拉丝                 |
| 排空体积（Blow-out Volume）               | 排液结束后再推出一小段空气，将吸头尖端残留液体“吹”出                     |
| 每次排液后添加尾随空气间隙                | 每次排液后重新吸入一段空气，重建样品后的空气屏障                         |

### 5.5 通道移液配置

![通道移液配置示意](media/image16.png)

#### 5.5.1 功能概述

通道移液测试是工程师调试模式下的核心功能，用于对指定液体通道（试剂位 R1~R40、样本位 S1~S8、清洗站等）进行完整的移液动作验证。支持手动或半自动执行吸液、排液、系统液冲洗、液面探测及通道清空等操作，用于校准移液精度、验证液路密封性及排查异常。

#### 5.5.2 测试要素定义

| **要素**                         | **说明**                                                                                       |
| -------------------------------- | ---------------------------------------------------------------------------------------------- |
| 孔位（Well Position）            | 指定待测试物理位置：试剂位（R1~R33，实际R1~R40）、样本位（S1~S8）、清洗站（内外壁）             |
| 液体量（Volume）                 | 用户输入的移液体积（μL），控制吸排液量                                                          |
| 吸液（Aspirate）                 | 从指定孔位吸取设定体积液体至移液针内                                                           |
| 打液 / 排液（Dispense）          | 将移液针内液体排出至指定孔位                                                                   |
| 打系统液（Flush with System Fluid） | 向通道注入系统清洗液（如去离子水或专用冲洗液），冲洗管路、防止交叉污染                          |
| 液面探测（Liquid Level Detection, LLD） | 在吸液前通过电容/压力等方式自动探测液面高度，确保吸头浸入深度合适，避免空吸或过深               |
| 清空通道（Purge / Clear Channel）| 强制将移液通道内残留液体完全排出，并可配合空气段吹扫，确保通道洁净                               |

### 5.6 混匀加热等控制

![混匀加热配置示意](media/image17.png)

#### 5.6.1 功能概述

- **混匀控制**：控制样品加样时的试剂混匀  
- **样本加热**：控制混匀模块上的加热装置  
- **试剂制冷**：控制试剂区制冷，包含制冷温度、当前温度、启动制冷等，具体参考通讯协议  
- **样本清洗**：通过蠕动泵控制（4 个泵对应 4 个通道），用户预设流量、启停时间，自动执行清洗流程  
- **排毒**：启动排毒泵，由脚本控制  
- **排废液**：启动排废液泵，由脚本控制

### 5.7 实验流程加载脚本控制

- 实验使用不同脚本配置，支持多个脚本同时运行

## 六、其他要求

6.1 **样本上样**：玻片可分次上载，单次最多 1~16 张，上一张染色过程中可加载新玻片且无需停机；每张玻片可设置不同染色程序。

6.2 **多批次优化**：当有多批次切片上载时，软件自动优化染色时间和程序，穿插执行各批次切片，实现染色时间最优化。

6.3 **染色程序**：支持 HE、冰冻免疫组化、特染及多重染色等多种程序。

6.4 **染色时间**：单片运行时间 ≤ 10 min；16 张满负荷染色 ≤ 15 min（**重要**）。

6.5 可连接医院 LIS 系统。

6.6 **后台数据**：记录机械臂运动正常及异常情况、玻片运行情况、试剂使用情况等，支持查询与统计。

6.7 每张片子可选取相应脚本开始实验（常规 IHC 和 HE 染色）。

6.8 **实验脚本可视化**：创建实验脚本的界面可视化，方便操作。

6.9 **UI 美观简洁**（初步示意图见下，需重新设计更美观）：

- 界面上显示的通道和实验舱通道一一对应  
- 每个通道有 4 张片子，每张可独立选脚本  
- 颜色显示通道状态，与机器上指示灯颜色一致  
- 通道下方显示实验完成比例  
- 选中通道时图标放大凸出，深色边框方便识别  
- 空闲通道玻片名称和选择框显示灰色，有玻片的显示深色  
- 选择 IHC 脚本后显示“IHC”，选择 HE 脚本显示“HE”  
- 通道拉出时界面上对应图标闪烁提醒  
- 4 个通道独立控制，互不干扰（空闲或已完成可拉出，不影响其他）  
- 机械臂有单独的控制文件 SDK（详见附件）  
- 下位机通讯协议详见附件
