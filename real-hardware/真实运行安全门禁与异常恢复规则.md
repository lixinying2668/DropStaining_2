## 2. 总体安全原则

> 以下原则为不可妥协项，覆盖所有真实运行阶段；任一违反即视为门禁失败。

### 2.1 核心原则

1. **未知即不安全**：任何状态、极性、阈值、映射、结果无法确认时，一律按不安全处理。
2. **未确认即不放行**：门禁项未现场确认或未通过评审，不得解除对应级别门禁。
3. **断连即 fail-closed**：任何关键通讯断连（主控 / SOCON / CAN / DCR55 / 制冷 / Bridge / IPC）立即进入安全保持。
4. **Pending / Unknown 不得自动重试**：设备动作结果为 Pending 或 Unknown 时，不得自动补发或自动继续。
5. **前端重试不得导致重复真实动作**：前端/SignalR 重连、用户重复点击、页面刷新不得触发重复真实物理动作。
6. **数据库异常不得导致重复真实动作**：数据库锁、写入失败、WAL 异常不得造成真实动作重复发送。
7. **Bridge 或设备异常不得自动恢复真实动作**：Bridge 崩溃、异常退出、SDK 运行期异常后，不得自动恢复或重发已下发动作。
8. **真实动作前必须具备**：CommandId、Pending 记录、审计、资源锁和状态回读策略；缺一不可。

### 2.2 放行与恢复原则

9. **保守恢复**：恢复必须从受控大步骤起点、人工确认点或工程处置流程开始，不得从断点静默续跑。
10. **资源不静默释放**：未知状态下针内液体、DAB、库存预约、资源锁不得被自动清除或释放。
11. **人工确认留痕**：所有人工解除、恢复、工程处置进入审计与运行追溯。
12. **分级递进**：真实能力按「只读 → 低风险写入 → 回零 → 低速空跑 → 吸液/加液 → 替代液验证 → 湿实验」递进，不允许跳级。
13. **资料 ≠ 现场**：「✅ 已确认（资料）」项不得当作现场已验证；协议默认值、SDK 默认值（如 NodeID、SetMaxTrip）严禁当作现场真实值。

### 2.3 与既有系统规则的一致性

- 本规则与 `PROJECT_CONTEXT.md` 第 2.5 节「真实设备动作前的最低门禁」、第 9.2 节「写入与设备通信规则」保持一致并细化。
- 本规则不削弱任何既有 fail-closed 边界；若存在冲突，以更保守者为准，并在评审中记录。

### 3.2 门禁求值规则

- 门禁求值为**逻辑 AND**：任一项失败即整体 fail-closed。
- 门禁求值必须在**动作下发前**原子完成，并记录求值结果（含每一项状态）到审计。
- 门禁求值失败不得静默重试真实动作；仅允许重新求值（只读复核）。
- 「待现场确认」项在未现场确认前，按**不通过**处理。

### 3.3 门禁分层与分级关系

- 本章为**总门禁**（最高层）；第 8 章在此基础上按能力级别定义**分级门禁**。
- 总门禁全部通过是分级门禁的**必要非充分**条件；分级门禁可追加额外前置（如 P0-05 校准、安全 SOP、替代液批准等）。

### 4.2 安全 IO 通用规则

- **断线默认不安全**：任一安全 IO 断线、超时、极性无法判定时，按其「触发条件已满足」处理（即不安全）。
- **立即阻止**：一级安全 IO 触发立即阻止后续真实动作，并在可能范围内安全保持当前状态。
- **只读不阻断**：标注「允许继续只读」者，表示该信号异常不禁止已获授权的只读状态查询；但不解除真实动作门禁。
- **恢复需复核**：所有恢复必须先**只读复核**设备状态，并由规定确认人确认，记录审计后方可解除联锁。
- **阻塞关系**：标注「阻塞 P1-xx」者，表示该 IO 未现场确认前，对应 P1 阶段不得放行。
- 辅助执行器（排水泵 IO 0、排毒泵 IO 1）由主控自动控制；其真实触发条件、与液位联锁的真实时序仍待现场确认，未确认前不得开放写入。

## 5. 通讯异常规则

> 本章覆盖真实通讯链路各类异常。核心：**不得自动重发真实物理动作；不得因前端/数据库异常重复发送动作；Pending/Unknown 保守处理；恢复前先只读复核**。

### 5.1 通讯异常清单与处置

| 编号 | 异常类型 | 检测方式（待现场确认具体阈值） | 处置 | 是否自动重发真实动作 | 恢复前要求 |
|:----:|:----|:----|:----|:----|:----|
| C01 | 超时 | 命令发出后在「待现场确认」超时内无应答 | 标记该 CommandId 为 Timeout/Pending；报警；阻止后续真实动作 | 否 | 只读复核设备状态 |
| C02 | CRC 错误 | CRC 校验失败 | 丢弃帧；标记通讯异常；报警；阻止后续真实动作 | 否（仅可重试只读查询，不重试真实动作） | 只读复核 |
| C03 | ACK 缺失 | 应答帧缺失 | 同 C01 超时处理 | 否 | 只读复核 |
| C04 | NAK | 收到 NAK | 标记命令 Failed；报警；不得自动重发真实动作 | 否 | 查明 NAK 原因、复核 |
| C05 | 异常帧 | 帧格式/长度不符 | 丢弃；标记异常；报警 | 否 | 只读复核、必要时厂商支持 |
| C06 | 分包 / 拼包异常 | 重组失败/越界 | 丢弃当前重组；标记异常；报警 | 否 | 只读复核 |
| C07 | 串口断连 | 串口不可用/错误 | 进入 fail-closed；报警；阻止后续动作 | 否 | 重连（受控）、人工确认、只读复核 |
| C08 | CAN 断连 | CAN 总线/Bridge 断连 | 进入 fail-closed；报警；禁止所有机械臂动作 | 否 | Bridge 重启（受控）、人工确认、位置/状态只读复核 |
| C09 | Named Pipe 断连 | IPC 断连 | 主进程标记 Bridge 不可用；阻止真实动作 | 否 | Bridge 恢复、IPC 自检、人工确认、只读复核 |
| C10 | Bridge 进程异常退出 | 进程退出/崩溃检测 | 主进程标记 Bridge 不可用；所有 Pending 动作保留为 Unknown；禁止自动恢复 | 否（严禁） | Bridge 重启、人工确认、逐项只读复核、Pending 处置 |
| C11 | SDK 元数据验证失败 | 反射/元数据加载失败 | 拒绝真实连接；报警 | 否 | 修复 SDK 部署、人工确认 |
| C12 | SDK 运行期异常 | 运行期抛出异常 | 标记动作 Failed/Unknown；报警；阻止后续动作 | 否 | 只读复核、人工确认 |
| C13 | IPC 请求超时 | IPC 超时 | 标记该请求 Pending/Timeout；阻止后续真实动作 | 否 | Bridge 健康、只读复核 |
| C14 | 设备动作结果 Unknown | 结果不可知（断连/超时/崩溃期间） | 保守保留为 Unknown；**不得**自动重试或自动视为成功/失败 | 否（严禁） | 工程处置：人工只读复核设备真实状态、对账数据库、决定继续/回退 |

### 5.2 通讯异常铁律

1. **不得自动重发真实物理动作**：无论超时、CRC、断连、崩溃，均不得自动重发已下发的真实动作。
2. **前端重试不重复**：前端/SignalR 重连触发的重试必须以 CommandId 幂等，不得产生新的真实动作。
3. **数据库异常不重复**：数据库锁、写入失败不得导致真实动作重复发送；数据库异常时动作结果保守为 Unknown。
4. **Pending / Unknown 保守**：一律保留状态、进入工程处置，等待人工只读复核与决定。
5. **恢复前只读复核**：任何通讯异常恢复前，必须先以只读方式复核设备真实状态（位置、温度、液位、安全 IO、Pending 对账）。

| R13 | 制冷状态未知 | 制冷 Unknown | 不自动制冷写入/续跑 | 工程师：制冷复核 | 工程处置流程 |

### 6.2 恢复通用规则

1. **未知不清除**：未知状态不得自动清除为已知「安全」。
2. **资源不释放**：未知状态下资源锁、针内液体、DAB、库存预约不自动释放。
3. **不自动继续**：未知状态下不得自动继续运行。
4. **恢复起点**：恢复必须从受控大步骤起点、人工确认点或工程处置流程开始。
5. **工程处置路径**：每一类中断必须明确「谁可处置、处置步骤、复核条件、审计要求」。
6. **全程留痕**：所有处置进入审计与运行追溯（含操作人、时间、原因、复核结果）。

### 6.3 工程处置流程（草案）

1. 进入 fail-closed / 安全保持。
2. 工程师介入，**只读复核**设备真实状态（安全 IO、位置、温度、液位、Pending）。
3. 对账数据库与设备真实状态，标记不一致项。
4. 按 R01–R14 规定路径处置（人工核对/重配/清洗/释放/回退）。
5. 必要时现场负责人 / 厂商工程人员介入。
6. 复核通过后，由规定确认人解除联锁，记录审计。
7. 从受控大步骤起点或工程处置终点恢复运行。

- **验收留痕**：每级验收形成正式记录（含异常清单）。
- **报警与审计**：每级均接入正式报警、审计、SignalR 与工程诊断。
- **随时可降级**：出现异常、未通过门禁即降级/退出至安全保持。
- **退出条件**：明确每级「完成」与「退出/回退」条件，不得模糊续跑。
